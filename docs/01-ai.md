# 01 - LLM API 层：lib/ai

## 概要

`lib/ai` 是 pi-moonbit 的基础包，对应 pi-mono 的 `@mariozechner/pi-ai`。
本阶段实现核心类型定义、模型抽象、流式事件系统和 Provider 注册机制。

## pi-mono ai 包分析

原始 TypeScript 包包含 34 个文件（10 核心 + 17 Provider 实现 + 7 工具），
支持 24+ LLM 提供商和 500+ 模型定义。本阶段聚焦于**类型基础**和**架构骨架**，
具体 Provider 实现留待后续。

### 核心抽象

| 概念 | TypeScript | MoonBit |
|------|-----------|---------|
| 消息 | `UserMessage \| AssistantMessage \| ToolResultMessage` | `enum Message` |
| 内容块 | `TextContent \| ThinkingContent \| ImageContent \| ToolCall` | `enum ContentBlock` |
| 停止原因 | `"stop" \| "length" \| "toolUse" \| "error" \| "aborted"` | `enum StopReason` |
| 流事件 | `AssistantMessageEvent` (discriminated union) | `enum AssistantMessageEvent` |
| 模型 | `Model<TApi>` interface | `struct Model` |
| API 标识 | `type Api = string & {}` | `struct ApiId(String)` |
| Provider 标识 | `type KnownProvider = string` | `struct ProviderId(String)` |
| 工具 | `interface Tool` | `struct Tool` |
| 上下文 | `interface Context` | `struct Context` |
| 流选项 | `interface StreamOptions` | `struct StreamOptions` |
| Provider 注册 | `registerApiProvider()` + map | `register_provider()` + Array |

## 关键设计决策

### 1. 开放标识符：newtype → 单字段 struct

pi-mono 用 `string & {}` 实现开放联合（允许任意字符串值作为 API/Provider 标识符）。
MoonBit 中用单字段 struct 实现：

```moonbit
pub(all) struct ApiId(String) derive(Eq, Hash, Show)
pub(all) struct ProviderId(String) derive(Eq, Hash, Show)
```

优势：类型安全（不会混淆 ApiId 和 ProviderId），同时保留开放性（任意字符串值）。

### 2. 可见性：`pub(all)` 策略

所有公开类型使用 `pub(all)` 可见性，确保下游包（agent、coding_agent 等）
可以自由构造这些类型。MoonBit 的 `pub` 仅允许读取，不允许外部构造。

### 3. 流式 API：回调模式

pi-mono 使用 `AsyncIterable<AssistantMessageEvent>`。MoonBit 的 async 支持尚不成熟，
本阶段采用回调模式：

```moonbit
pub(all) struct EventHandler((AssistantMessageEvent) -> Unit)

pub fn stream(model, context, options, handler) -> Unit!StreamError
```

`EventHandler` 是一个包装了回调函数的单字段 struct。Provider 通过调用
`handler.0(event)` 推送事件。待 MoonBit async 成熟后可迁移为 channel/iterator 模式。

### 4. Provider 注册表：函数式风格

与 pi-mono 一致，不使用 OOP trait 而是直接存储 stream 函数：

```moonbit
register_provider(api_id, stream_fn)
get_provider(api_id) -> stream_fn?
```

注册表用 `Array[ProviderEntry]` 实现（线性查找，API 数量少无性能问题）。

### 5. `complete()` 基于 `stream()`

`complete()` 是 `stream()` 的便捷包装，收集所有事件直到终止事件，返回最终的
`AssistantMessage`。

## 与 pi-mono 的差异

| 方面 | pi-mono | pi-moonbit |
|------|---------|-----------|
| Provider 实现 | 17 个内置实现 | 仅类型和注册机制（实现留待后续） |
| 模型数据库 | 500+ 模型自动生成 | 仅已知常量（按需添加） |
| OAuth | 完整 OAuth 流程 | 暂不实现 |
| 验证 | TypeBox + AJV | 暂不实现（待 agent 阶段） |
| 溢出检测 | 19 种 Provider 错误模式 | 暂不实现 |
| 懒加载 | `createLazyStream()` | 直接注册（无需懒加载） |

## 测试覆盖

- **provider_test.mbt**: Provider 注册/查找、未注册 API 错误、stream 委托、complete 收集

共 15 个测试，全部通过。
